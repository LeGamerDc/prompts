# 需求文档模板与质量自检

生成最终文档前，先执行质量自检清单，再按模板生成文档并保存到 `docs/plan/Requirement.md`。

---

## 质量自检清单

### 必要性检查
- [ ] 问题是否具体可感知
- [ ] 问题是否有事实依据
- [ ] 影响范围是否清晰

### 逻辑性检查
- [ ] 目标与问题有明确因果关系
- [ ] 方法与目标匹配
- [ ] 整体逻辑链条完整

### 可行性检查
- [ ] 范围在合理的资源和时间内可完成
- [ ] 技术方案可实现
- [ ] 依赖条件可满足

### 可验收性检查
- [ ] 交付物明确具体
- [ ] 验收标准可操作
- [ ] 检查点覆盖完整

如有检查项未通过，需要与用户进一步沟通补充。
