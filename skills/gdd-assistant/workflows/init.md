# 项目初始化工作流

## 触发条件

`gdd/` 目录不存在，或用户明确要求初始化新策划项目。

## 流程

### 1. 了解项目

通过对话了解以下信息（用户不确定的部分可跳过，后续用创意模式探索）：

- 项目名称和一句话描述
- 游戏类型 / 核心玩法方向
- 目标受众
- 设计支柱（2-4 个，"这个游戏最看重什么？"）
- 核心情感体验（"希望玩家感受到什么？"）
- 参考作品（"有哪些启发来源？"）

### 2. 创建目录结构

```
gdd/
├── memory/
│   ├── project.md
│   └── systems/
├── vision/
│   ├── overview.md
│   ├── pillars.md
│   └── core-loop.md
├── systems/
└── shared/
    ├── glossary.md
    └── conventions.md
```

### 3. 填充初始文档

**`vision/overview.md`**：

```markdown
# <项目名>

## 一句话描述
<用一句话概括这个游戏是什么>

## 游戏类型
<类型和子类型>

## 目标受众
<目标玩家画像>

## 核心情感体验
<希望玩家感受到什么>

## 参考作品
- <作品名>: <借鉴什么>
```

**`vision/pillars.md`**：

```markdown
# 设计支柱

所有设计决策的评判标准。每个系统设计都必须服务于至少一个支柱。

## 支柱 1: <名称>
<说明：这个支柱意味着什么，在设计中如何体现>

## 支柱 2: <名称>
<说明>

（2-4 个为佳，过多会失去聚焦）
```

**`vision/core-loop.md`**：

```markdown
# 核心循环

## 主循环
<描述玩家重复进行的核心行为链>

## 驱动力
<是什么驱使玩家持续循环：好奇心/成长/收集/社交/...>

## 进度感
<玩家如何感知自己在进步>

## 单次循环时长
<一个循环大约多久，节奏如何>
```

**`memory/project.md`**：

```markdown
# 项目 Memory

## 状态
- 阶段: 概念
- 最后更新: <today>

## 进度概览
（初始化完成，尚无系统设计）

## 关键决策
- [<today>] 项目启动，初始化策划工程

## 跨系统约定
（暂无）

## 用户偏好
（待观察记录）

## 待解决（全局）
- 确定需要哪些核心系统
- 细化核心循环设计
- 完善设计支柱的具体含义

## 回顾
- [<today>] 项目初始化完成
```

**`shared/glossary.md`**：创建空模板，标题 + "随设计推进填充"。

**`shared/conventions.md`**：创建空模板，标题 + "随设计推进填充"。

### 4. 引导下一步

初始化完成后，建议用户：
1. 先完善 vision 文档（信息不足可进入创意模式讨论）
2. 确定需要设计的核心系统列表
3. 按优先级排序，从最核心/最明确的系统开始设计
