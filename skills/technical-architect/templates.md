# 文档模板与质量自检

生成最终文档前，先执行质量自检清单，再按模板生成文档。技术方案保存到 `docs/plan/Technical.md`，决策记录保存到 `docs/plan/DecisionRecords.md`。

---

## 质量自检清单

### 需求对齐检查
- [ ] 技术方案是否完整覆盖需求范围
- [ ] 是否引入了需求范围之外的工作
- [ ] 技术选型是否支持需求中的非功能性要求

### 约束一致性检查
- [ ] 所有选型是否在约束空间内
- [ ] 选型之间是否互相兼容
- [ ] 是否存在未满足的硬约束

### 架构合理性检查
- [ ] 组件职责是否单一清晰
- [ ] 组件依赖是否无环且方向合理
- [ ] 数据的生产消费关系是否完整闭合

### 任务可执行性检查
- [ ] 每个任务是否有明确的验收条件
- [ ] 任务依赖是否形成 DAG
- [ ] 任务粒度是否适中（不过大不过碎）

### 阶段可校验性检查
- [ ] 每个阶段是否有独立验证点
- [ ] 工作量与阶段时间是否匹配
- [ ] 高风险验证是否安排在早期
- [ ] 关键路径是否已识别

### 风险覆盖检查
- [ ] 高不确定性的技术点是否已识别
- [ ] 每个高风险项是否有验证计划和失败预案
- [ ] 验证任务是否已纳入阶段规划

如有检查项未通过，需要与用户进一步沟通补充。
