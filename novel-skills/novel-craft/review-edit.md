# 审阅编辑方法论

## 触发场景

- 用户要求审阅章节、检查一致性、修订正文
- 章节写作完成后进入审稿闭环
- 用户要求整体质量检查（跨多章）

## 前置条件

- 有待审阅的章节正文（`production/drafts/chapters/`）
- 项目 `production/reviews/checklist.md` 已存在

## 工作流

采用**自治模式**：agent 按清单自主完成审阅，输出审阅报告和返工建议。

### 第一步：确定审阅范围

- **单章审阅**：写完一章后的常规审稿
- **批量审阅**：完成一卷或一批章节后的整体检查
- **定向审阅**：用户指出特定问题，针对性检查

### 第二步：四类审稿

参照项目 `production/reviews/checklist.md`，逐项检查：

#### 结构审稿
- 开头是否快速进入局面（不超过 2 段铺垫）
- 本章是否有明确推进（信息/关系/资源/局势至少一项变化）
- 中段是否有灌水（无推进的段落）
- 结尾是否有钩子（悬念/反转/期待/情感/信息）
- 节奏是否松紧交替（不是全程紧张或全程平淡）

#### 设定审稿
- 人物称呼、身份、关系是否与角色卡一致
- 时间线是否合理（几天前/几个月前的事件是否对得上）
- 地点与行动路径是否合理（距离、地形）
- 体系规则、能力、资源是否前后冲突
- 与 `canon/outline/foreshadowing.md` 和 `index/unresolved-threads.md` 是否矛盾

#### 风格审稿
- 是否白、顺、易读（无过度文艺化）
- 是否有过度抒情或解释（用事件而非说明传达信息）
- 爽点是否表达清楚（不藏太深）
- 是否存在重复表达
- 对话是否符合角色语音标记

#### 桥段审稿（如使用了桥段卡）
- 桥段是否服务人物动机和剧情（不是为聪明而聪明）
- 桥段的 sequence 是否在正文中完整呈现
- 是否与既有设定冲突

### 第三步：输出审阅报告

审阅结果写入 `production/reviews/auto-reviews/` 或 `consistency-checks/`，格式：

```
## 审阅报告：vol-{NN}/ch-{NNN}

### 结构
- [通过/问题] 具体说明

### 设定
- [通过/问题] 具体说明

### 风格
- [通过/问题] 具体说明

### 返工建议
1. [返工类型] 具体修改建议
2. ...

### 总评
一句话评价 + 是否需要返工
```

### 第四步：返工

如需修改，按五种返工类型针对性处理：

| 类型 | 场景 | 操作 |
|------|------|------|
| **提速** | 节奏拖沓 | 删冗余说明和无效对话，强化事件推进 |
| **增钩** | 章末平淡 | 加强悬念、反转或信息钩 |
| **强爽点** | 爽感不足 | 把获得、反制、成长的反馈写得更明确、更有冲击 |
| **补逻辑** | 设定冲突 | 修复因果链、补齐信息缺口、修正时间线 |
| **调文风** | 风格偏移 | 更口语、更顺读、更贴平台调性 |

返工原则：
- 只针对指定问题修改，不整章无目标重写
- 修改后重新检查受影响的审稿项
- 如修改导致设定变更，走"设定修改"路由（改 canon → 同步 index → 补 ops）

### 第五步：人工审稿入口

自动审稿完成后，提醒用户进行人工审稿：
- 这章好不好看？
- 是否想追更？
- 最无聊的部分在哪？
- 最有劲的部分在哪？
- 哪个信息点/冲突点值得在下一章放大？

人工反馈记入 `production/reviews/human-notes/`。

## 跨章审阅

完成一卷或 10+ 章时，执行整体检查：

1. **节奏曲线**：每章的紧张度是否有起伏，是否存在连续多章平淡
2. **伏笔审计**：`foreshadowing.md` 中是否有长期未推进的伏笔
3. **角色弧线**：主要角色是否在成长或变化，是否停滞
4. **设定一致性**：跨章的时间线、距离、能力是否连贯
5. **阅读体验**：如果一个新读者从第一章开始，能否被吸引持续读下去

## 回写清单

1. 审阅报告 → `production/reviews/auto-reviews/` 或 `consistency-checks/`
2. 返工任务 → `production/reviews/revision-tasks/` 或 `ops/todo.md`
3. 人工反馈 → `production/reviews/human-notes/`
4. 如发现设定错误 → 修改 `canon/` 对应文件并同步 `index/`
5. 更新 `ops/status.md`

## 下一步

审阅通过后：
- 继续写作 → [chapter-writing.md](chapter-writing.md)
- 如需补充设定 → [worldbuilding.md](worldbuilding.md) 或 [characters.md](characters.md)
- 如需调整大纲 → [outline.md](outline.md)
