# 大纲规划方法论

## 触发场景

- 用户要求规划大纲、情节结构、分卷、伏笔
- 项目已完成基础设定，需要建立剧情骨架
- 正文写作前需要细化章节 beats

## 前置条件

- `core/high-concept.md` 已填充（有故事引擎和成长路线）
- 至少有主角的角色卡

## 工作流

采用**协作模式**。大纲是人机协作最密集的阶段——agent 提出结构建议，用户确认和调整。

### 四级规划体系

```
总大纲 → 卷大纲 → 剧情弧 → 章节 Beats
  L1        L2       L3        L4
```

逐级细化，上一级确认后再展开下一级。

### 第一步：总大纲（L1）

创建 `canon/outline/master-outline.md`，使用 [templates.md](templates.md) 的总大纲模板：

1. **总主线**：一段话概括整个故事
2. **阶段划分**：拆成 3-6 个大阶段，每个阶段有明确的核心目标和转折点
3. **终局设想**：故事最终走向（可标注"待定"）

参考 `core/high-concept.md` 的长线成长路线来划分阶段。

结构模型选择（建议在 `ops/decisions.md` 中记录选了哪个）：
- **三幕结构**：设置 → 对抗 → 解决
- **起承转合**：起（破局） → 承（积累） → 转（危机） → 合（突破）
- **英雄之旅**：日常 → 召唤 → 历险 → 归来
- **网文爽文结构**：新手 → 崛起 → 争霸 → 巅峰（每个阶段内循环：挑战 → 突破 → 收获 → 更大挑战）
- **自定义**：根据题材需要自由组合

### 第二步：卷大纲（L2）

为每卷创建 `canon/outline/volumes/vol-{NN}.md`，使用卷大纲模板：

1. **卷目标**：本卷要解决的核心问题
2. **起点/终点状态**：主角和世界在本卷前后的对比
3. **关键事件**：按顺序列出里程碑
4. **新角色/新设定**：如有
5. **预计章数和字数**

卷的划分原则：
- 每卷有独立的主矛盾（卷内闭环）
- 卷末留悬念连接下一卷
- 第一卷最关键——前 20 章决定读者去留

### 第三步：剧情弧（L3）

为主线和重要支线创建 `canon/outline/arcs/` 下的弧文件，使用剧情弧模板：

1. **目标**：这条弧要完成什么
2. **节拍**：设置 → 升级 → 危机 → 高潮 → 解决
3. **跨卷范围**：起止卷号
4. **涉及角色**

弧的类型：
- **主线弧**：贯穿全书的核心冲突
- **卷弧**：单卷内的主要矛盾
- **角色弧**：特定角色的成长线
- **感情弧**：感情线的发展

弧与弧之间要有交错——A 弧的高潮期 B 弧在铺垫，形成持续的阅读驱动力。

### 第四步：章节 Beats（L4）

为即将写作的章节创建 beats（`canon/outline/chapter-beats/`），使用章节 beats 模板：

1. **本章目标**：这一章要推进什么
2. **Beats 列表**：场景/事件 — 涉及角色 — 叙事功能
3. **必须满足**：不能违反的设定点
4. **钩子方向**：章末留什么悬念

Beats 不需要一次规划太远。建议保持 5-10 章的 beats 储备，写作过程中滚动补充。

## 伏笔矩阵

伏笔是长篇连载的核心体验之一。在 `canon/outline/foreshadowing.md` 中维护伏笔矩阵。

### 伏笔生命周期

```
埋设 (planted) → 发展 (developing) → 收回 (paid-off)
```

### 管理规则

1. **埋设时**：在矩阵中新增一行，标注埋设章节和内容
2. **发展时**：追加发展章节，记录伏笔如何被强化或暗示
3. **收回时**：标注收回章节，更新状态为 paid-off
4. **定期审计**：每完成一卷，检查是否有遗忘的伏笔

伏笔矩阵格式见 [templates.md](templates.md)。

### 伏笔设计原则

- 大伏笔（跨卷）控制在 5-8 个以内，避免线头太多
- 每个伏笔的埋设要自然，不能为伏笔而伏笔
- 收回时要有情感冲击，不能草草收场
- 伏笔状态变更时同步更新 `index/unresolved-threads.md`

## 回写清单

大纲工作后：
1. 创建或更新 `canon/outline/` 下的对应文件
2. 同步 `index/canon-index.md` 的剧情结构部分
3. 新增伏笔记入 `canon/outline/foreshadowing.md` 和 `index/unresolved-threads.md`
4. 更新 `ops/status.md` 和 `ops/todo.md`
5. 新增 `ops/handoffs/` 交接记录

## 下一步

至少完成第一卷的卷大纲和前 5-10 章的 beats 后：
- [chapter-writing.md](chapter-writing.md) — 开始写作
- 如发现需要补充设定 → [worldbuilding.md](worldbuilding.md) 或 [characters.md](characters.md)
