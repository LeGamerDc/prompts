# 设定调研方法论

## 触发场景

- 构思阶段确定了历史、架空历史或特定行业背景，需要调研真实世界基础
- worldbuilding 过程中发现对某个社会维度缺乏认知（如"宋代到底有没有棉布？"）
- 写作过程中遇到需要真实细节支撑的场景（如"角色去市集买东西，该用什么货币？"）
- 开始一部新的历史/架空题材小说前，需要全景了解目标时代

## 与 plot-research.md 的区别

| | plot-research.md | setting-research.md（本文档） |
|---|---|---|
| 回答的问题 | **怎么写**（叙事手法、桥段套路、情节结构） | **写什么**（真实世界的社会、历史、领域知识） |
| 典型素材源 | TV Tropes、写作博客、经典文学 | Wikipedia、古籍数据库、学术搜索 |
| 输出格式 | 桥段卡、技巧卡 | 设定事实卡、时代速写卡 |
| 存放位置 | `research/plot/` | `research/setting/` |

## 工作流

采用**自治模式**：agent 自主搜索和整理，结果写入项目的 `research/setting/` 目录。

## 调研维度框架

对任何故事背景，以下九个维度构成完整的社会画像。不需要全部调研——根据题材选择重点维度。

| 维度 | 核心问题 | 种田/经营 | 权谋/历史 | 都市/行业 | 军事 |
|------|---------|:---------:|:---------:|:---------:|:----:|
| **政治结构** | 谁统治？权力如何分配？决策流程？ | ○ | ★ | ○ | ★ |
| **社会阶层** | 阶层划分？流动性？身份标识？ | ★ | ★ | ○ | ○ |
| **经济运作** | 货币？贸易？产业？物价？ | ★ | ★ | ★ | ○ |
| **科技水平** | 工具？工艺？交通？通信？医疗？ | ★ | ○ | ○ | ★ |
| **军事体系** | 编制？武器？战术？兵役？ | — | ★ | — | ★ |
| **日常生活** | 饮食？服饰？住宅？出行？娱乐？ | ★ | ○ | ★ | ○ |
| **文化风俗** | 信仰？节日？礼仪？艺术？教育？ | ○ | ○ | ○ | — |
| **职业体系** | 哪些职业？社会地位？日常内容？ | ★ | ○ | ★ | ○ |
| **地理环境** | 地形？气候？资源？交通要道？ | ★ | ○ | — | ★ |

★ = 必须深入调研 ○ = 按需了解 — = 通常不涉及

### 维度展开参考

每个维度的调研子项，帮助避免遗漏：

**政治结构**
- 中央政体（皇权/共和/议会…）与地方行政层级
- 官制：官职名称、品级、选拔方式（科举/世袭/推举）
- 权力制衡机制
- 法律制度与司法流程
- 税收体系

**社会阶层**
- 阶层划分标准（血统/财富/功名/职业）
- 各阶层的权利与限制
- 阶层流动途径（科举/经商/军功/联姻）
- 身份外在标识（服饰/住宅/称呼/礼仪差异）

**经济运作**
- 货币制度（实物/金属/纸币，兑换比率）
- 主要产业与贸易路线
- 商业组织形式（行会/商帮/公司）
- 典型物价（一餐/一匹布/一亩地/一间房）
- 赋税与劳役

**科技水平**
- 农业技术（耕作方式、灌溉、育种）
- 手工业技术（冶金、纺织、陶瓷、造纸、印刷）
- 交通工具与速度（步行/骑马/马车/船/铁路）
- 通信手段与速度（驿站/飞鸽/电报）
- 医疗水平（能治什么、不能治什么）
- 建筑技术与典型建筑形制

**军事体系**
- 军队编制（从最小单位到最高指挥层级）
- 武器装备（攻击/防御/攻城/水战）
- 兵员来源（征兵/募兵/世兵）
- 后勤补给方式
- 战术与阵法常识

**日常生活**
- 一日三餐（或两餐）的内容、饮食禁忌
- 各阶层服饰特征与穿着规范
- 住宅形制与室内陈设
- 出行方式与旅行条件
- 娱乐活动（节庆/游戏/戏曲/体育）
- 卫生与生活习惯

**文化风俗**
- 主流宗教/信仰体系
- 重要节日与仪式
- 婚丧嫁娶礼仪
- 社交礼节与禁忌
- 教育制度（官学/私塾/书院）
- 主流文学艺术形式

**职业体系**
- 当时存在的主要职业门类
- 各职业的社会地位与收入水平
- 职业的日常工作内容与节奏
- 行业术语与行话
- 入行门槛与晋升路径

**地理环境**
- 核心地理特征（山川/河流/海岸/平原）
- 气候特点与季节变化
- 关键资源分布（矿产/林木/水源/农田）
- 主要交通路线与关隘
- 城市布局与规模

## 素材源

### Tier 1：直接抓取

| 素材源 | 核心价值 | 获取方式 | URL 模式 |
|--------|---------|---------|---------|
| **Wikipedia (EN)** | 系统性最强的历史/社会条目，有 category tree 可层级遍历 | `WebFetch` | `https://en.wikipedia.org/wiki/{Article}` |
| **中文维基百科** | 中国历史特有条目，如官制、科举、地名沿革 | `WebFetch` | `https://zh.wikipedia.org/wiki/{条目}` |
| **中国哲学书电子化计划** | 古籍原文全文检索，考据一手资料 | `WebFetch` | `https://ctext.org/zh` |

### Tier 2：搜索引擎中介

| 素材源 | 核心价值 | 获取方式 | 搜索模式 |
|--------|---------|---------|---------|
| **百度百科** | 中文语境的快速概述，适合获取中文名词解释 | `WebSearch` → `WebFetch` | `site:baike.baidu.com {关键词}` |
| **知乎** | 历史爱好者的深度讨论和科普 | `WebSearch` 间接获取 | `site:zhihu.com {关键词}` |
| **学术搜索** | 特定领域的考证和专题研究 | `WebSearch` | `{关键词} 研究 PDF` 或 `{keyword} history journal` |
| **专题网站** | 特定领域的深度内容（如古建筑、古代服饰） | `WebSearch` → `WebFetch` | 按需搜索 |

### Wikipedia 深度使用技巧

Wikipedia 是设定调研最核心的素材源。以下是高效使用方法：

1. **从总览条目出发**：先读时代/领域的主条目（如 `Song dynasty`），获取整体框架
2. **按维度追踪内链**：主条目中会链接到政治、经济、科技等子条目，逐个深入
3. **利用 category tree**：条目底部的分类可以发现同类相关条目
4. **中英对照**：英文条目通常更系统，中文条目有本地化细节，两者互补
5. **注意时代边界**：某些技术和制度有明确的出现时间，要记录"从何时开始"和"到何时消失"

## 搜索策略

### 策略 A：全景扫描

**场景**：刚确定时代/领域背景，需要快速建立整体认知

1. `WebFetch` Wikipedia 主条目（如 `Song dynasty`）
2. 从主条目提取各维度的子条目链接
3. 按优先维度（参考维度框架的 ★ 标记）逐个 `WebFetch`
4. 整理为一张**时代速写卡**
5. 标记需要深入调研的维度

```
示例搜索链路：
需求：「了解宋朝社会全貌」
→ WebFetch: en.wikipedia.org/wiki/Song_dynasty → 提取框架
→ 追踪子条目: Economy_of_the_Song_dynasty, Technology_of_the_Song_dynasty,
   Society_of_the_Song_dynasty, Song_dynasty_in_warfare
→ WebFetch: zh.wikipedia.org/wiki/宋朝 → 补充中文语境
→ 输出: 宋代全景速写卡
```

### 策略 B：维度深钻

**场景**：某个维度需要深入了解（如"宋代经济系统"）

1. `WebFetch` 该维度的 Wikipedia 专题条目
2. 追踪条目内的关键内链（如"交子" → "会子" → "市舶司"）
3. `WebSearch` 补充中文深度内容：`site:zhihu.com {维度} {时代}`
4. 对关键事实进行交叉验证
5. 整理为多张**设定事实卡**

```
示例搜索链路：
需求：「深入了解宋代货币与商业」
→ WebFetch: en.wikipedia.org/wiki/Economy_of_the_Song_dynasty
→ 追踪: Jiaozi_(currency), Huizi_(currency), Maritime_trade_of_the_Song_dynasty
→ WebFetch: zh.wikipedia.org/wiki/交子 → 具体发行时间、面额、使用范围
→ WebSearch: site:zhihu.com 宋代商业 坊市制 → 补充夜市、瓦子等细节
→ 输出: 宋代货币事实卡、宋代商业形态事实卡
```

### 策略 C：细节补充

**场景**：写作中遇到具体问题（如"宋朝人吃什么主食"）

1. `WebSearch` 直接搜索具体问题
2. `WebFetch` 最相关的 1-2 个结果
3. 如有矛盾信息，追加搜索验证
4. 记录为简短的**设定事实卡**

```
示例搜索链路：
需求：「宋朝普通人的一日三餐」
→ WebSearch: 宋朝 饮食 日常 平民
→ WebFetch: 最相关结果
→ WebSearch: Song dynasty daily meals commoners （英文交叉验证）
→ 输出: 宋代饮食事实卡（含"宋代已有一日三餐"等关键事实）
```

### 策略 D：职业/身份调研

**场景**：需要了解特定职业的细节以塑造角色

1. `WebSearch` + `WebFetch` 该职业的 Wikipedia 条目
2. 关注：日常工作内容、社会地位、收入水平、行业术语、入行门槛
3. `WebSearch` 补充文学作品或历史记载中该职业的描写
4. 整理为**设定事实卡**，重点标注可用于写作的细节

```
示例搜索链路：
需求：「宋代铁匠的日常生活」
→ WebSearch: 宋代 铁匠 手工业 锻造
→ WebFetch: en.wikipedia.org/wiki/Blacksmith + Song dynasty technology
→ WebSearch: 宋代 官营手工业 民间手工业 铁器
→ 输出: 宋代铁匠事实卡（含工具、工序、产品类型、社会地位、收入估算）
```

### 策略 E：交叉验证

**场景**：某个关键事实可能影响剧情走向，需要确认准确性

1. 在至少两个独立来源中查找同一事实
2. 英文源和中文源各至少一个
3. 如有矛盾，标注争议，取最保守（最不易出错）的版本
4. 在事实卡中标注可信度等级

## 输出格式

### 时代速写卡

对一个时代或领域的全景概述，作为后续深入调研的路线图。存放于 `research/setting/`。

```markdown
# {时代/领域}速写

## 基本信息
- **时间跨度**: {起止年份}
- **地理范围**: {疆域/区域}
- **人口规模**: {大致数量}
- **调研日期**: {YYYY-MM-DD}

## 政治结构
{2-3 句概述}
- 需深入: {列出待深入的子话题}

## 社会阶层
{2-3 句概述}
- 需深入: {待深入子话题}

## 经济运作
{2-3 句概述}
- 需深入: {待深入子话题}

## 科技水平
{2-3 句概述}
- 需深入: {待深入子话题}

## 日常生活
{2-3 句概述}
- 需深入: {待深入子话题}

## 文化风俗
{2-3 句概述}
- 需深入: {待深入子话题}

## 职业体系
{主要职业门类列举}
- 需深入: {待深入子话题}

## 标志性特征
{这个时代/领域最具辨识度的 5-10 个特征，可直接用于写作}

## 常见误区
{容易写错的地方，如把后世才有的东西写进去}

## 来源
{列出参考的主要 URL}
```

### 设定事实卡

记录单个经过验证的事实或事实集合。存放于 `research/setting/`。

```markdown
## 事实：{主题}

- **时代/领域**: {所属时代或领域}
- **维度**: {政治/经济/科技/社会/军事/日常/文化/职业/地理}
- **可信度**: 高（多源一致） | 中（单源但权威） | 低（仅见于非学术源，待验证）
- **内容**:
  {结构化的事实描述，含关键数据和细节}
- **写作可用细节**:
  {直接可以在小说中使用的具体细节，如器物名称、价格、颜色、尺寸等}
- **常见错误**:
  {关于这个话题容易犯的时代错误}
- **来源**: {URL 列表}
```

## 与 worldbuilding 的衔接

1. 设定调研产出的事实卡存放在 `research/setting/`，是**外部参考**，不是 canon
2. worldbuilding 过程中引用事实卡，将真实世界知识**转化**为故事设定
3. 架空设定可以有意偏离真实历史，但需要在 canon 中明确标注偏离点
4. 事实卡中的"常见错误"帮助避免无意识的时代错误

## 注意事项

1. **先广后深**：先用策略 A 全景扫描，再按需用策略 B 深钻，避免一上来就钻牛角尖
2. **Wikipedia 条目可能很长**（50-200KB），优先读引言和目录，按需展开章节
3. **中文维基百科部分条目较薄**，历史类条目建议中英对照
4. **知乎只能通过 WebSearch 间接获取**，直接 WebFetch 会超时
5. **ctext.org 适合考据**，但古文需要额外理解成本，只在需要一手材料时使用
6. **架空不等于可以乱来**——即使是架空历史，读者对基础常识有预期，科技树不能乱点
7. **标注时间边界**——记录事实时务必注明"从何时开始"、"到何时结束"，避免张冠李戴
8. **事实卡只是参考**——真正采用的内容要转写进 `canon/lore/`，不能让正文直接依赖 research
