# 世界观构建方法论

## 触发场景

- 用户要求设计世界观、地点、势力、体系
- 项目已完成构思阶段，需要资产化设定
- 正文写作中发现需要补充世界设定

## 前置条件

- 项目已初始化（有 AGENTS.md）
- `core/positioning.md` 和 `core/high-concept.md` 已填充（至少有基础方向）
- **如果基于真实历史或特定行业背景**：建议先走 [setting-research.md](setting-research.md) 搜集事实基础，产出时代速写卡和设定事实卡，再进入本流程。已有的事实卡会让世界观构建更准确、细节更扎实。

## 工作流

采用**协作模式**：与用户讨论确认后再写入 `canon/lore/` 下的对应文件。

### 第一步：确定世界框架

先建立宏观世界观（`canon/lore/world/`），按 [templates.md](templates.md) 的世界观条目格式。如果已通过 [setting-research.md](setting-research.md) 完成调研，参考 `research/setting/` 中的事实卡填充各项：

1. **时代与背景**：什么时代、什么世界？现实历史 / 架空 / 异世界？
2. **核心规则**：这个世界与现实的关键差异（如果有）
3. **社会结构**：政治体制、阶层关系、权力分配
4. **技术/文明水平**：决定角色的行动边界和日常细节
5. **氛围基调**：这个世界给人的整体感受

原则：宏观框架确定后尽量稳定，后续只在子层级扩展。

### 第二步：题材专属体系

根据 `core/positioning.md` 的主类型，设计核心体系（`canon/lore/systems/`）：

**种田/经营体系**需要定义：
- 可种植/养殖/生产的品类及其产出周期
- 资源获取与消耗链
- 升级路径（产量提升、品质进化、规模扩张）
- 经济循环（生产 → 加工 → 交易 → 投资）
- 金手指的边界与代价（如有）

**修仙/玄幻体系**需要定义：
- 等级划分和每级的标志性能力
- 修炼资源和瓶颈
- 战力对比规则
- 宗门/势力的等级对应
- 天赋/功法/法宝的体系

**其他类型**按需设计核心体系，原则相同：明确层级、资源、限制和驱动冲突的机制。

关键：体系要为剧情服务。每个机制要能产出冲突或爽点，纯设定堆砌无意义。

### 第三步：地点设定

按需建立地点条目（`canon/lore/locations/`），使用 [templates.md](templates.md) 的地点模板：

- 起始地点：主角开局所在地，需要最详细的描写
- 核心场景：前几卷中反复出现的地点
- 远景地点：后续卷会涉及的地点，先占位标注即可

每个地点要回答：这里有什么叙事功能？（冲突场、安全屋、交易点、探索区…）

### 第四步：势力与组织

按需建立势力条目（`canon/lore/factions/`），使用 [templates.md](templates.md) 的势力模板：

- 明确每个势力对主角的态度和功能（提供压力？提供资源？提供盟友？）
- 势力之间的博弈关系
- 势力与世界体系的联系

### 第五步：经济与风俗

如果故事涉及资源流转、交易、社会互动：
- `canon/lore/economy/`：货币、物价、交易规则、稀缺资源
- `canon/lore/customs/`：影响角色行为的社会规范和习俗

### 回写清单

每次新增或修改设定后：
1. 同步更新 `index/canon-index.md`
2. 如有未定事项，记入 `index/unresolved-threads.md`
3. 更新 `ops/status.md` 和 `ops/todo.md`
4. 新增 `ops/handoffs/` 交接记录

## 一致性原则

- 世界规则一旦确定，后续不可随意变更。如需修改，走"设定修改"任务路由（改 canon → 同步 index → 补 ops）
- 地点的距离、环境、资源要前后一致
- 体系的等级、能力、限制不能想起一出是一出
- 所有硬性规则在文档中用粗体或列表明确标注
