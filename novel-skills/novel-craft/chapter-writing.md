# 章节写作方法论

## 触发场景

- 用户要求写章节、继续写、写下一章
- 用户要求写特定场景
- 按生产工作流进入写作阶段

## 前置条件

- 项目已初始化且有完善的 AGENTS.md
- `core/` 下的规则文件已填充
- 当前章节有对应的 beats（`canon/outline/chapter-beats/`）
- 主要出场角色有角色卡

如缺少 beats，先引导用户通过 [outline.md](outline.md) 补充。

## 工作流

采用**自治模式**：agent 按流程自主完成写作，写后按回写规则更新文件。

### 第一步：上下文加载

严格遵循项目 `core/context-assembly.md` 的五层注入和拼装顺序：

1. **恒定层** — 读取 `core/positioning.md` + `core/style-guide.md` + `core/quality-bar.md`
2. **全局剧情层** — 读取 `core/high-concept.md` + 当前卷大纲
3. **局部上下文层** — 读取：
   - 本章 beats（`canon/outline/chapter-beats/`）
   - 最近 1-3 章的正文或摘要
   - 出场角色的角色卡
   - `index/unresolved-threads.md` 中与本章相关的线头
   - `ops/handoffs/` 最新 1-2 份交接
4. **桥段层**（如有） — 本章要用的桥段卡（1-2 张）
5. **检索层** — 本章涉及的地点、体系、势力等设定

### 第二步：写前校验

在开始写作前检查：

- [ ] 上一章结尾状态：人物在哪、在做什么、什么情绪
- [ ] 本章出场角色的当前状态和关系
- [ ] 本章涉及的设定点（地点距离、资源数量、能力限制等）
- [ ] 本章需要推进的剧情弧和伏笔
- [ ] 本章不能违反的硬性设定

这一步是**防崩设定**的关键。常见错误：角色名字写错、遗忘上一章的伤/情绪状态、距离和时间不合理、能力突然升级。

### 第三步：场景写作

按 beats 逐场景写作。每个场景遵循：

**场景结构：**
- **开场**：快速建立时间、地点、角色状态（不超过 2 段）
- **发展**：事件推进、冲突展开、信息交换
- **收束**：场景结果 + 过渡到下一场景的桥接

**写作技法检查清单：**
- [ ] 感官细节：至少两种感官（视觉 + 听觉/触觉/嗅觉）
- [ ] 对话有潜台词：说出来的和真正想说的不完全一样
- [ ] 角色语音一致：对话符合角色卡中的语音标记
- [ ] 内心活动适度：不过多也不缺失，用于揭示角色动机
- [ ] 节奏控制：紧张场景用短句短段，舒缓场景可以展开
- [ ] 信息密度：每段都在推进（情节/情感/关系/信息），删除纯填充内容

**场景间转换：**
- 时间跳跃用 `---` 分隔
- 地点转换在新场景开头一句话交代
- 避免大段总结叙述（"接下来几天，主角做了很多事"）

### 第四步：章节收尾

**结尾必须有钩子**，类型包括：
- 悬念钩：引出新问题或新威胁
- 反转钩：刚获得的东西被打破或颠覆
- 期待钩：主角做出决定，读者想知道结果
- 情感钩：关系出现新变化
- 信息钩：透露一个让读者好奇的线索

**章节 frontmatter**（在正文文件顶部）：

```yaml
---
volume: 卷号
chapter: 章号
title: 章标题
pov: 视角角色
location: 主场景
word-count: 实际字数
status: draft
---
```

### 第五步：写后回写

完成一章后执行以下回写（不可跳过）：

1. **保存正文** → `production/drafts/chapters/vol-{NN}/ch-{NNN}.md`
2. **更新章节 beats** → 标记对应 beats 为已完成，记录实际偏差
3. **更新伏笔矩阵** → 本章埋设/发展/收回的伏笔，在 `canon/outline/foreshadowing.md` 中更新状态
4. **更新 unresolved-threads** → 新增的线头加入，已解决的标记闭环
5. **更新 ops/status.md** → 当前进度
6. **更新 ops/todo.md** → 勾选已完成项，添加下一章待办
7. **新增 handoff** → `ops/handoffs/YYYY-MM-DD-HHMM-ch{NNN}.md`，记录本章写作的关键信息

如果本章的角色状态发生了重大变化（受伤、获得新能力、关系转变），还需更新对应角色卡。

## 连续写作模式

连续写多章时：
- 每章之间执行完整的写后回写
- 开始新章节前重新执行上下文加载（避免前一章的临时上下文污染）
- 每 3-5 章做一次简要审查：整体节奏是否合理、是否有遗忘的伏笔

## 常见质量问题

| 问题 | 检测方法 | 修复 |
|------|---------|------|
| 崩设定 | 核对角色卡和世界观 | 回溯修改，更新 canon |
| 灌水 | 检查每段是否有推进 | 删除纯填充，强化事件 |
| 钩子弱 | 章末是否有悬念 | 增加反转或信息钩 |
| 节奏失控 | 全章是否只有一个调 | 调整场景节奏的松紧交替 |
| 人物同质化 | 对话是否能区分角色 | 强化语音标记 |

## 下一步

章节写完后：
- [review-edit.md](review-edit.md) — 审阅和修订
- 继续下一章 → 回到本文档第一步
