# 构思阶段方法论

## 触发场景

- 用户要求开始写一部新小说
- 用户要求设计故事设定或故事引擎
- 用户要求修改项目定位
- 用户进入已初始化但 `core/positioning.md` 尚未填充的项目

## 前置：项目初始化

如果项目尚未初始化（没有 `AGENTS.md`）：
1. 与用户确认项目根目录路径和小说标题
2. 读取 [scaffold.md](scaffold.md)，按其流程创建完整目录结构和骨架文件
3. 完成后继续本流程

## 工作流

采用**协作模式**：引导讨论，等用户确认后再写入文件。

### 第一步：平台与题材定位

引导用户明确以下要素，写入 `core/positioning.md`：

| 要素 | 说明 |
|------|------|
| 目标平台 | 番茄小说 / 起点中文网 / 晋江 / 其他 |
| 主类型 | 种田 / 玄幻 / 都市 / 悬疑 / 言情 / 历史 / 科幻 / 其他 |
| 目标读者 | 画像和阅读偏好 |
| 核心卖点 | 3-5 条，按重要性排列 |
| 差异化方向 | 与同类作品的区别 |
| 写作禁忌 | 本作不做什么 |
| 作品关键词 | 5-10 个标签 |

可参考下方的类型速查表获取各类型的常用卖点和调性。

### 第二步：故事引擎设计

引导用户设计故事的核心驱动机制，写入 `core/high-concept.md`：

1. **一句话故事**：用一句话概括整个故事
2. **三句话故事**：开局处境 → 核心行动 → 最终走向
3. **核心爽点**：读者持续追读的 3-5 个动力源
4. **核心问题**：主角为什么必须行动？优势是什么？外部压力来自哪里？
5. **长线成长路线**：阶段性目标链（如"生存 → 立足 → 扩张 → 博弈 → 称霸"）
6. **剧情推进轮子**：2-4 个持续产出剧情的循环机制

推进轮子的设计是网文长篇连载的关键——每个轮子独立转动又相互咬合，确保剧情永远有事发生。

### 第三步：文风适配

确认或调整 `core/style-guide.md` 的默认设定：
- 视角和时态
- 对话密度
- 描写风格
- 平台特定的风格约束

### 第四步：状态更新

完成构思后：
1. 更新 `ops/status.md`：阶段从"0"推进到"1：构思完成，进入设定阶段"
2. 更新 `ops/todo.md`：添加下一步待办
3. 将关键决策记录到 `ops/decisions.md`
4. 新增一份 `ops/handoffs/` 交接记录

## 类型速查表

常见中文网文类型的默认调性和卖点参考。用户选择类型后，以此为起点定制。

### 种田 / 经营
- 调性：稳中有爽、日常温暖
- 卖点：资源积累、生活改善、建设成就、把烂摊子盘活
- 推进轮子：生产轮 + 关系轮 + 外部压力轮 + 成长轮
- 典型金手指：系统面板、穿越知识、空间、植物亲和

### 玄幻 / 修仙
- 调性：热血爽快、等级碾压
- 卖点：修炼突破、宝物获取、打脸装逼、宗门攀升
- 推进轮子：修炼轮 + 探索轮 + 势力轮 + 机缘轮
- 典型金手指：功法传承、老爷爷、特殊体质、系统

### 都市
- 调性：贴近现实、逆袭爽感
- 卖点：事业成功、能力展示、身份反转、情感满足
- 推进轮子：事业轮 + 社交轮 + 对手轮 + 感情轮
- 典型金手指：重生记忆、透视、医术、系统

### 悬疑 / 推理
- 调性：紧张烧脑、层层反转
- 卖点：解谜快感、真相揭露、智力碾压、反转冲击
- 推进轮子：线索轮 + 嫌疑人轮 + 危机轮 + 真相轮
- 典型结构：案件串联、大案套小案、双线叙事

### 言情 / 古言
- 调性：甜虐交替、情感满足
- 卖点：双向奔赴、感情升温、误会化解、身份揭秘
- 推进轮子：感情轮 + 误会轮 + 外部阻力轮 + 成长轮
- 典型金手指：重生、穿越、读心、空间

### 历史 / 架空
- 调性：厚重写实（或轻松穿越）
- 卖点：权谋博弈、历史改写、文明建设、知识碾压
- 推进轮子：政治轮 + 军事轮 + 经济轮 + 人才轮
- 典型金手指：穿越带现代知识、系统、记忆
